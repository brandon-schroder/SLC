## Confirmed — the loading sign convention (a *positive* verification)

K-O's cascade convention writes the secondary-loss loading and mean angle as
- `C_L/(s/c) = 2(tan α1 + tan α2)·cos α_m`   (a **sum**)
- `tan α_m = ½(tan α1 − tan α2)`              (a **difference**)

The code, in its unified *signed* meridional frame (`:142–158`), writes
- `load = 2·|tan α2 − tan α1|·cos_m`  (a **difference**)
- `tan_m = ½(tan α1 + tan α2)`         (a **sum**)

These are **the same physics**. For a turbine row whose inlet/exit swirl sit
on opposite sides of meridional, K-O's magnitude-angle convention (α1, α2
both positive on opposite sides) maps to the code's signed convention by a
sign flip on one angle — which swaps sum↔difference in exactly these two
places. Worked check (α1=+30°, α2=−60° signed ≡ 30°/60° K-O): loading
2.309 and tan α_m −0.577 come out identical both ways. **This confirms the
`kacker_okapuu.py:145–146` "frame-safe in signed cascade angles" claim** —
the sum/difference swap is not a bug, it is what makes the transcription
frame-invariant.

## Findings — code deviates from KO82 (documented, not silently changed)

1. **Profile-loss interpolation weight (negative incidence) — RESOLVED
   2026-07.** K-O-1982 modified AM-1957's zero-incidence weight `(β1/α2)²` to
   the **signed** `|β1/α2|·(β1/α2)` "to account for negative inlet angles." The
   code had used the AM-1957 *symmetric* `r²`. Since the module *is* the
   Kacker-Okapuu set (every other term follows K-O, incl. the thickness
   exponent K-O flipped from AM), the target is KO82: `profile_loss_am` now
   uses `abs_smooth(r)·r`. **Behavior-preserving for β1≥0** (identical to `r²`),
   so every in-domain case is untouched — V6 runs `r=β1/α2 ∈ [0.04, 0.72]`
   (measured), all positive. For `r<0` the signed weight pulls loss *below*
   nozzle (the negative-incidence bucket) instead of AM's symmetric rise. A C¹
   positivity floor (`smooth_max` at ½·nozzle-loss) keeps the surrogate curves
   from returning negative loss under deep-negative extrapolation (AD-10; a
   safety, not a calibration — no in-domain case reaches it). Pinned physics-
   anchored in `tests/test_kacker_okapuu_reference.py::
   test_profile_weight_is_ko82_signed_not_am_symmetric`.

2. **Shock hub-to-tip factor.** K-O scale the hub shock loss to the mean line
   by `(R_hub/R_tip)`, and evaluate `M1` at the **hub** (highest U). The code
   (`:206`) already flags both as `[VERIFY]`-deferred and evaluates the local
   relative Mach per streamtube instead. Consistent with the code's own
   deferral note; unchanged.

## Confirmed — the trailing-edge ζ→Y conversion

`axial_turbine/loss.py:127` maps the K-O trailing-edge **kinetic-energy** loss
coefficient `ζ` (= `Δφ²_TE`) to an equivalent total-pressure loss `Y` via
`Y_TE = ζ/(1−ζ)` before summing into the single B.3 exit-referenced `Y`. The
source gives the **exact compressible** relation (verbatim):

    Y = { [1 − (γ−1)/2·M2²·(1/φ² − 1)]^(−γ/(γ−1)) − 1 }
        / { 1 − (1 + (γ−1)/2·M2²)^(−γ/(γ−1)) }

whose `M2 → 0` incompressible limit is `Y = Δφ²/(1 − Δφ²)`, i.e. exactly the
coded `ζ/(1−ζ)`. ✅ **Confirmed as the incompressible-limit form** (pinned in
`tests/test_kacker_okapuu_reference.py`). The Mach-dependent version is the
recorded refinement — `φ²` varies weakly with M2 vs `Y`, so the limit is a
reasonable simplification for the (small) TE term.

## Profile-loss curves — DIGITIZED & recalibrated (2026-07)

The nozzle/impulse profile-loss **reference curves** `yp1`,`yp2` were the
code's smooth surrogate fits to the Ainley-Mathieson chart. They are now
**calibrated to digitized points off R&M 2974 Fig. 4** (Fig. 4a nozzle β1=0,
Fig. 4b impulse β1=−α2; `t/c=20%`, `Re=2×10⁵`, `M<0.6`). K-O use these AM
curves unchanged for the AMDC term (their 2/3, `K_p`, `f_Re` recalibration is
applied *outside* `profile_loss_am`), so R&M 2974 Fig. 4 is the right source.

- **Method** (`tools/digitize_am_fig4.py`, rerunnable): each labelled curve's
  minimum was read against the printed grid, cross-checked against an
  automated dark-pixel column scan (gridline detection + envelope trace) and
  the canonical Dixon / CRS values. Reading precision ≈ ±0.005 in Y_p.
- **Calibrated form**: minimum-loss level `A + B·u⁴` (`u=α2/70`; the data is
  flat-then-steep — u⁴ beats u² by ~2× on residual), optimum-pitch line
  linear in u, single symmetric-parabola curvature. Reproduces the digitized
  minima to **<0.003** (nozzle) / **<0.005** (impulse) in Y_p — the old
  invented constants over-predicted the level by ~0.010. Pinned in
  `tests/test_kacker_okapuu_reference.py::test_{nozzle,impulse}_curve_matches_am_fig4*`.
- **Documented residual**: the chart curves are *asymmetric* (steep for
  s/c<s_opt, flat for s/c>s_opt) and their curvature grows with exit angle;
  the symmetric, angle-independent parabola is a compromise accurate near the
  loading optimum where rows operate, over-predicting only at simultaneously
  high α2 AND high s/c.

**[BUG] found & fixed (2026-07) — positivity-floor width was angle-scaled.**
The AD-10 positivity floor `smooth_max(bracket, ½·yp1, width)` reused the
**angle-ratio** width `_R_W=0.1`. `smooth_max` over-estimates by up to
`width·ln2`, so a 0.1 width on **loss-scale** values (Y_p~0.02–0.15) silently
inflated *every* profile loss by ~0.06 (→ ~0.037 in the assembled profile-Y
after the 2/3·0.914 factors). The old surrogate hid it inside V6's structural
bands. Fixed to a loss-scaled `_YP_FLOOR_W=0.003`; regression
`test_profile_loss_floor_width_is_loss_scaled_not_angle_scaled`. V6 stays
in-band (PR 0.932→0.934, inverted-η 1.072→1.043, i.e. *less* loss, correct
direction — profile loss is small for modern K-O blades; secondary/shock
dominate).
