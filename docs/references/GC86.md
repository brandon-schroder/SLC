# GC86 — Gallimore & Cumpsty (1986) spanwise mixing coefficient

Sources: Gallimore & Cumpsty, "Spanwise Mixing in Multistage Axial Flow
Compressors" (ASME J. Turbomach. / *Trans.* 1986) — **primary**, present in the
NotebookLM "Reduced-Order Aerodynamic Solvers" notebook; and Wisler, Bauer &
Okiishi (1987), "Secondary Flow, Turbulent Diffusion, and Mixing…" — a
secondary evaluation, present in the "Loss Models" notebook. Cross-checked
against `slcflow/transport/mixing.py`.

**Provenance.** Extracted 2026-07-09 from those notebooks (source-grounded,
citation-backed).

## Confirmed — the model form

Gallimore-Cumpsty model spanwise mixing as a **random turbulent-diffusion**
process incorporated into a through-flow calculation. `mixing.py`'s parabolic
operator (§3.6) — implicit spanwise diffusion of `{h0, s, rVθ}` — is that form.
✅ (The primary excerpts confirm the turbulent-diffusion mechanism; the exact
diffused-property list `{h0,s,rVθ}` is the standard through-flow application,
not quoted verbatim from this excerpt.)

## Finding — the coefficient value AND its length scale both differ

Gallimore-Cumpsty's recommended non-dimensional diffusion coefficient (from
the primary paper):

- Definition: **`ε / (V_z · L_s)`**, where `V_z` = mean axial velocity through
  the stage and `L_s` = **axial stage length**.
- Value: measured slope `ε/(V_z L_s) = 1.8×10⁻³`; "high, about the same as a
  two-dimensional turbulent wake" (`2.7×10⁻³`); a simple predictive model gives
  `1.6×10⁻³` (larger machine) / `2.1×10⁻³` (smaller). The authors note "an
  estimate for ε good to one significant figure appears adequate." So the
  recommended level is **≈ 2×10⁻³** (range 1.6–2.7×10⁻³). Wisler's independent
  evaluation on the LSRC computed `1.75×10⁻³` via the G-C model (Adkins-Smith
  `2.57×10⁻³`) — consistent — and stresses the value is **configuration
  dependent**.

The code uses `mu_mix = c_mix · ρ · Vm · r` with **`c_mix = 0.01`**. Its
operator `d(χ)/dm = 1/(r(1−B)ρVm) · d/dq(mu_mix r dχ/dq)` reduces to an
effective spanwise diffusivity (diffusion length per unit m)

    D_code = mu_mix / ((1−B) ρ Vm) = c_mix · r / (1−B),

whereas the G-C form gives `ε/V_z ≈ 1.8×10⁻³ · L_s`. Matching the two:

    c_mix ≈ 1.8×10⁻³ · (L_s / r) / (1−B).

**Two mismatches, compounding:**
1. **Length scale.** The code nondimensionalizes on the local radius `r`; G-C
   use the axial stage length `L_s`. These are different scales (`L_s ≈
   0.05–0.1 m`, `r ≈ 0.2–0.5 m` for a typical stage → `L_s/r ≈ 0.1–0.5`).
2. **Magnitude.** Even after the length-scale factor, a G-C-consistent value is
   `c_mix ≈ 1.8×10⁻³·(L_s/r) ≈ 2×10⁻⁴ … 9×10⁻⁴` — so the coded `0.01` is
   **~10–50× stronger** than G-C's calibration.

Net: `mixing.py`'s default mixing is far more aggressive than Gallimore-Cumpsty
recommend. Because mixing is **off by default** (`FidelityConfig.mixing_term=0`
in all tiers) and only the multistage V5 case opts in, this bites only there —
but it means the M8 multistage stratification numbers were produced with an
over-strong mixing coefficient.

## Recommendation (`[DECIDE]`, not silently changed)

Two clean options, both needing a deliberate re-run of the multistage V5
stratification check:
- **(A) Re-base the operator on `L_s`** (axial stage length) and adopt G-C's
  value directly: `ε/(V_z L_s) = 1.8×10⁻³`. Matches the source's
  nondimensionalization exactly; needs the stage axial length in the mixing
  flow view.
- **(B) Keep the `r`-based form** and set `c_mix ≈ 1.8×10⁻³·(L_s/r)` for
  representative geometry (~`5×10⁻⁴`). Simpler, but `c_mix` then carries a
  geometry-dependent factor and is not a clean transcription of G-C.

Not changed here: `c_mix` stays `0.01` pending this decision and a paired
V5-stratification re-measurement (the M8 "25× stratification" claim is
sensitive to it). The `[VERIFY]` in `mixing.py` is sharpened to record this.
